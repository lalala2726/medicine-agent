# Analysis Skill（supervisor）

## 技能目标

在 `supervisor_agent` 中，针对“数据分析、运营复盘、趋势判断、可视化展示”类请求，执行一套稳定流程：

1. 识别需求范围（订单 / 商品 / 用户 / 运营 / 图表）
2. 调用对应子代理获取真实数据
3. 汇总并形成结论
4. 必要时输出图表配置代码块

该技能适用于需要“多工具联动”的分析场景，而非单纯闲聊。

## 可用工具与职责映射

- `order_tool_agent`：订单列表、订单详情（含地址/物流/明细）、订单流程时间线、发货记录
- `after_sale_tool_agent`：售后列表、售后详情（按售后申请 ID）
- `product_tool_agent`：商品列表、商品详情、药品详情
- `user_tool_agent`：用户列表/详情、用户钱包、钱包流水、消费信息
- `analytics_tool_agent`：运营总览、趋势、分布、排行、退货率
- `get_current_time`：需要“今日/本周/本月”等时间语义时先获取当前时间

这些工具都是一个独立的 agent 这些都可以使用自然语言进行调用，它们内部都有自己的工具
但是注意的是有些数据需要你从另一个工具获取到数据之后再传入另一个工具，常见的字段如 userId orderId productId 等

比如订单工具 这边能获取到订单的详情，但是商品详细信息这边没有，这边只有一个商品的基本信息，如果需要商品更加详细的信息，比如生产厂家，功效、说明书、这边你需要调用商品工具并且给对应的商品 ID

## 分析决策流程

### 第一步：识别用户意图

按请求内容归类并选择工具：

- 运营总览、趋势、分布、热销、退货率 -> `analytics_tool_agent`
- 订单筛选、订单详情、物流、收货信息、流程时间线、发货记录 -> `order_tool_agent`
- 售后筛选、售后详情、售后处理进度排查 -> `after_sale_tool_agent`
- 商品筛选、商品详情、药品说明 -> `product_tool_agent`
- 用户筛选、用户详情、钱包与消费排查 -> `user_tool_agent`
- 用户明确要图表，或结果适合可视化 -> 先取数，再根据 chart 技能提供的模板规范生成图表


### 第二步：处理参数与结构化约束

调用子代理时遵循这些硬约束：

- 趋势周期：`DAY` / `WEEK` / `MONTH`（未明确时默认 `DAY`）
- TopN：`limit` 范围 `1-200`
- 订单详情 `order_id` 必须是 `List[str]`
- 商品详情/药品详情 `product_id` 必须是 `List[str]`
- 用户详情/钱包/流水/消费查询使用单个 `user_id`，且必须大于 0
- 售后详情查询使用单个 `after_sale_id`，且必须大于 0
- 禁止使用逗号拼接字符串替代数组（如 `"1,2,3"`）

## 输出规范

### 文本输出

- 先给结论，再给关键证据数据。
- 优先使用表格/列表展示核心指标。
- 金额保留两位小数；大数可用万/亿单位。
- 不暴露内部调度过程（不要写“我调用了某某工具”）。

### 图表输出（当用户需要图表或数据明显适合可视化）

1. 必须参考 chart 技能提供的目标图表模板规范
2. 代码块语言标识必须是图表类型本身（如 `line`、`pie`、`bar`）
3. 代码块内容必须是合法 JSON
4. 数值字段使用纯数字，禁止带 `%`、`k`、`万` 等符号
5. 字段结构必须严格遵循模板，不新增自定义字段

## 异常与空数据处理

- 工具返回空数据：明确说明“当前条件下暂无数据”，并给出可执行的条件调整建议。
- 工具失败：如实反馈失败，并建议用户缩小范围或补充条件后重试。
- 用户问题条件不足：只追问最关键缺失项，避免过度追问。

## 典型任务模板

### 模板 A：运营复盘

1. 调用 `analytics_tool_agent` 获取总览 + 趋势 + 分布
2. 汇总关键变化（增长/下滑）
3. 输出“结论 + 指标表”
4. 若用户要求可视化，根据 chart 技能提供的模板规范输出图表配置

### 模板 B：问题定位（退货率高 / 订单异常）

1. 先用 `analytics_tool_agent` 定位异常指标
2. 按需联动 `product_tool_agent` 或 `order_tool_agent` 下钻
3. 输出“发现 -> 证据 -> 建议动作”

### 模板 C：跨域问答（订单 + 商品 + 图表）

1. 拆分为“取数子任务”
2. 逐个调用对应子代理
3. 统一口径（同时间范围、同统计维度）
4. 合并结论并给图表代码块

# 订单域 (Order) 专家指南

## 领域概述

`order_tool_agent` 负责商城订单全生命周期的管理与监控。它不仅是订单数据的源头，更是连接用户域与商品域的关键纽带。

---

## 核心能力与返回界限

### 1. 订单列表检索
- **用途**：按状态、支付方式、收货人信息等批量筛选订单。
- **返回**：订单概览（ID、编号、状态、金额、时间）。

### 2. 订单详情 (支持批量)
- **用途**：获取订单的最完整视图，包括收货地址和购买明细。
- **返回的关键 ID (用于跨域接力)**：
  - `userId`：下单用户的唯一标识 -> 转至 **User Agent**。
  - `productId`：订单内每个商品的唯一标识 -> 转至 **Product Agent**。

**⚠️ 数据边界（必读）**：
> 订单详情中的商品信息仅为 **“快照摘要”**（ID、名称、图片、单价、购买量）。
> **它不持有以下数据**：
> - 实时库存、商品分类、上下架状态。
> - 药品说明书、禁忌、用法用量等专业信息。
> **处理建议**：如用户询问上述信息，必须提取 `productId` 后二次调度 **Product Agent**。

### 3. 物流与发货审计
- **用途**：追踪物流轨迹时间线及具体发货记录（快递单号）。

---

## 典型编排场景

### 场景：排查“订单中商品的实时库存”
1. **Step 1 (Order)**: `order_tool_agent("查询订单 O20260101 的详情，明确要求返回 productId 列表")`。
2. **Step 2 (Product)**: 提取真实 ID 后调用 `product_tool_agent("查询商品 ID 为 [数值] 的详情，重点关注库存")`。

### 场景：核验“下单用户的钱包余额”
1. **Step 1 (Order)**: `order_tool_agent("查询订单 O20260101 的详情，获取 userId")`。
2. **Step 2 (User)**: 提取真实 ID 后调用 `user_tool_agent("查询用户 ID 为 [数值] 的钱包余额")`。

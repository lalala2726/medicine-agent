# 跨域协作操作手册

## 核心原则

**ID 是跨域查询的钥匙。** 当一个工具的返回结果中包含其他域的 ID（如 productId、userId、orderId），并且用户需要该域的详细信息时，必须主动调用对应工具补查，不能复用上一步已有的摘要信息。

### 各工具返回的跨域 ID 一览

| 来源工具                            | 返回结果中包含的跨域 ID                           | 说明                                                |
| ----------------------------------- | ------------------------------------------------- | --------------------------------------------------- |
| order_tool_agent（订单详情）        | productId（商品列表中）、userId（用户信息中）     | 商品只有 ID/名称/数量/单价，用户只有 ID/昵称/手机号 |
| after_sale_tool_agent（售后详情）   | orderId、orderNo、userId、productId（商品信息中） | 商品信息同样是摘要，用户只有 ID 和昵称              |
| analytics_tool_agent（排行/退货率） | 商品相关的 ID                                     | 只有统计汇总，不含商品经营和药品专业信息            |
| user_tool_agent（用户详情）         | 无直接跨域 ID，但用户信息可间接用于订单/售后查询  | 需要用用户信息作为筛选条件                          |

### 摘要 vs 详情对照

**订单/售后中的商品信息（摘要）只有 6 个字段**：

- 商品 ID、名称、图片、单价、数量、总价

**商品详情额外提供**：

- 库存、分类名称、上下架状态、配送方式、商品图片列表等

**药品详情额外提供**：

- 功效/适应症、用法用量、不良反应、禁忌、注意事项、成分、说明书全文、生产单位、批准文号、是否处方药等

## 场景 1：查订单中的商品详细信息

> **用户问法**："帮我查最新 10 个订单里每个商品的详细信息"、"这些订单的商品库存还有多少？"、"订单里面买的都是什么分类的商品？"

**操作步骤**：

1. 调用 `order_tool_agent("查询最新 10 个订单的详情")` → 获取订单列表及详情
2. 从返回结果中收集所有不重复的 **商品 ID**
3. 调用 `product_tool_agent("查询商品 ID 为 xxx、xxx、xxx 的详情")` → 获取库存、价格、分类等完整商品信息
4. 将订单中的购买记录与商品详细信息合并呈现

**关键关注**：第 3 步返回的库存数量、分类名称、上下架状态是订单中没有的。

---

## 场景 2：查订单中药品的禁忌和适应症

> **用户问法**："这些订单里的药有什么禁忌？"、"订单里买的药说明书帮我查一下"、"这些商品有没有处方药？"

**操作步骤**：

1. 调用 `order_tool_agent("查询订单 O20260101 的详情")` → 获取订单中的商品列表
2. 从商品列表中提取 **商品 ID**
3. 调用 `product_tool_agent("查询商品 ID 为 xxx、xxx 的药品详情，包括禁忌、适应症和用法用量")` → 获取药品专业信息
4. 按商品逐一展示药品信息

**关键关注**：药品的功效、禁忌、用法、不良反应、是否处方药都只有药品详情工具能提供。

---

## 场景 3：查订单买家的详细资料

> **用户问法**："下单的用户是谁？详细信息帮我看一下"、"这个订单的买家资料"、"订单的用户什么时候注册的？"

**操作步骤**：

1. 调用 `order_tool_agent("查询订单 O20260101 的详情")` → 获取订单中的用户信息
2. 从用户信息中提取 **用户 ID**
3. 调用 `user_tool_agent("查询用户 ID 为 xxx 的详情")` → 获取完整用户资料
4. 合并订单中的下单信息和用户详细资料一起汇报

**关键关注**：订单中的用户信息只有 ID、昵称、手机号，不含邮箱、身份证、注册时间、登录记录、消费统计等。

---

## 场景 4：查订单买家的钱包和消费

> **用户问法**："下单用户的钱包还有多少钱？"、"看看买家的消费及钱包情况"

**操作步骤**：

1. 调用 `order_tool_agent("查询订单 O20260101 的详情")` → 获取用户 ID
2. 调用 `user_tool_agent("查询用户 ID 为 xxx 的钱包信息")` → 获取钱包余额和状态
3. （可选）调用 `user_tool_agent("查询用户 ID 为 xxx 的消费信息")` → 获取消费记录
4. 汇总钱包和消费情况

---

## 场景 5：售后对应的订单详情

> **用户问法**："这个售后对应的订单什么情况？"、"售后单关联的订单详情看一下"、"售后订单什么时候发的货？"

**操作步骤**：

1. 调用 `after_sale_tool_agent("查询售后 ID 为 30001 的详情")` → 获取售后信息（含 orderId / orderNo）
2. 从结果中提取 **订单 ID 或订单编号**
3. 调用 `order_tool_agent("查询订单 xxx 的详情")` → 获取订单完整信息
4. （可选）如需物流信息，继续调用 `order_tool_agent("查询订单 xxx 的发货记录")`
5. 将售后信息和订单信息整合呈现

---

## 场景 6：售后商品的完整信息和说明书

> **用户问法**："售后涉及的商品详细信息看一下"、"退货的这个药说明书帮我查查"、"售后商品的库存和价格"

**操作步骤**：

1. 调用 `after_sale_tool_agent("查询售后 ID 为 30001 的详情")` → 获取售后中的商品 ID
2. 调用 `product_tool_agent("查询商品 ID 为 xxx 的商品详情")` → 获取库存、价格、分类
3. （如需药品信息）调用 `product_tool_agent("查询商品 ID 为 xxx 的药品详情")` → 获取说明书、禁忌、用法
4. 合并售后信息 + 商品信息 + 药品信息

---

## 场景 7：售后用户的账号状态和消费记录

> **用户问法**："提售后的这个人是什么情况？"、"是不是新注册的号？消费多少？"

**操作步骤**：

1. 调用 `after_sale_tool_agent("查询售后 ID 为 30001 的详情")` → 获取 userId
2. 调用 `user_tool_agent("查询用户 ID 为 xxx 的详情")` → 获取注册时间、总订单数、总消费等
3. （可选）调用 `user_tool_agent("查询用户 ID 为 xxx 的消费信息")` → 获取消费明细
4. 综合判断用户资质并汇报

---

## 场景 8：用户买过什么药、这些药有什么注意事项

> **用户问法**："用户 1001 买过什么药？这些药有什么注意事项？"、"看看他买的药有没有禁忌冲突"

**操作步骤**：

1. 调用 `user_tool_agent("查询用户 ID 为 1001 的消费信息")` → 获取消费记录
2. 从消费记录中获取关联订单信息，调用 `order_tool_agent("查询对应订单的详情")` → 获取商品 ID 列表
3. 收集所有不重复的商品 ID，调用 `product_tool_agent("查询商品 ID 为 xxx、xxx 的药品详情")` → 获取注意事项、禁忌
4. 按药品逐一汇总注意事项

**关键关注**：这是一个三步跨域链路（用户 → 订单 → 药品），前两步主要是为了获取商品 ID。

---

## 场景 9：热销商品的库存检查

> **用户问法**："卖得最好的几个商品库存还够不够？"、"热销前 5 的商品看看库存情况"

**操作步骤**：

1. 调用 `analytics_tool_agent("查询热销商品排行 TOP5")` → 获取热销商品 ID 和销量数据
2. 收集返回结果中的 **商品 ID**
3. 调用 `product_tool_agent("查询商品 ID 为 xxx、xxx、xxx 的详情，重点关注库存")` → 获取库存数量
4. 对比销量和库存，给出库存是否充足的分析

---

## 场景 10：退货率高的商品问题排查

> **用户问法**："退货率最高的几个商品有什么问题？看看说明书"、"退货多的药品是不是有什么副作用？"

**操作步骤**：

1. 调用 `analytics_tool_agent("查询退货率排行 TOP5")` → 获取退货率高的商品 ID 和退货数据
2. 调用 `product_tool_agent("查询商品 ID 为 xxx、xxx 的药品详情，关注不良反应和注意事项")` → 获取药品专业信息
3. （可选）调用 `product_tool_agent("查询商品 ID 为 xxx、xxx 的商品详情")` → 获取价格、库存
4. 综合退货率数据和药品信息分析可能的原因

---

## 场景 11：完整排查一笔售后（全链路）

> **用户问法**："帮我全面排查一下售后 30001，包括订单、用户、商品所有信息"

**操作步骤**：

1. 调用 `after_sale_tool_agent("查询售后 ID 为 30001 的详情")` → 获取售后基础信息、orderId、userId、productId
2. 调用 `order_tool_agent("查询订单 xxx 的详情")` → 获取订单完整信息（含收货地址、支付信息）
3. 调用 `user_tool_agent("查询用户 ID 为 xxx 的详情")` → 获取用户完整资料
4. 调用 `product_tool_agent("查询商品 ID 为 xxx 的商品详情")` → 获取库存、价格
5. 调用 `product_tool_agent("查询商品 ID 为 xxx 的药品详情")` → 获取说明书
6. 汇总五部分信息，形成完整的售后排查报告

**关键关注**：这是最完整的全链路查询，覆盖售后→订单→用户→商品→药品全部维度。

## 场景 12：对比两个订单的商品差异

> **用户问法**："帮我对比一下这两个订单分别买了什么，价格和库存都看看"

**操作步骤**：

1. 调用 `order_tool_agent("批量查询订单 O20260101 和 O20260102 的详情")` → 获取两个订单的商品列表
2. 分别收集两个订单中的 **商品 ID**（去重）
3. 调用 `product_tool_agent("查询商品 ID 为 xxx、xxx、xxx 的详情")` → 获取所有涉及商品的完整信息
4. 按订单维度对比展示商品差异（名称、数量、价格、库存、分类）

---

## 场景 13：用户投诉全面排查

> **用户问法**："用户 1001 投诉了，帮我看看他的情况，订单、售后都查一下"

**操作步骤**：

1. 调用 `user_tool_agent("查询用户 ID 为 1001 的详情")` → 了解用户基本情况和消费统计
2. 调用 `after_sale_tool_agent("查询用户 ID 为 1001 的售后列表")` → 看有哪些售后申请
3. 如有待处理售后，调用 `after_sale_tool_agent("查询售后 ID 为 xxx 的详情")` → 了解具体问题
4. 调用 `order_tool_agent("查询相关订单详情")` → 了解订单情况
5. 综合分析用户投诉原因并汇报

---

## 场景 14：商品在哪些订单中被购买

> **用户问法**："商品 2001 最近被哪些订单买了？"

**操作步骤**：

1. 目前无法直接按商品 ID 筛选订单列表（接口不支持）
2. 可以引导用户换个角度查询，例如：
   - 使用 `analytics_tool_agent` 查看该商品的销量数据
   - 使用 `after_sale_tool_agent` 查看该商品是否有售后记录
3. 如实告知用户当前不支持按商品反查订单，建议从订单列表中搜索

## 通用操作原则

1. **先判断是否需要跨域**：用户只要问到"详细""完整""更多"或者明确提及另一个域的信息，就需要跨域补查
2. **传 ID 要准确明确**：调用下游工具时，在自然语言描述中写清具体 ID 值（如"查询商品 ID 为 2001、2003 的详情"）
3. **去重后批量查询**：多个订单可能包含相同商品，收集商品 ID 时先去重，避免重复查询
4. **信息合并时标注来源**：汇总时按订单/商品/用户等维度清晰组织，用表格或列表呈现
5. **不要复用旧数据**：当上下文中已有某些数据但用户要求"详细信息"时，仍需重新调用工具获取最新完整数据

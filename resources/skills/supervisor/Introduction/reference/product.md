# 商品工具使用指南

## 工具概述

`product_tool_agent` 负责所有商品相关查询，内部包含 3 个工具，接收自然语言任务描述。

---

## 工具能力与返回数据

### 1. 商品列表

**用途**：按条件搜索商品，支持分页浏览。

**可用筛选条件**：商品 ID（精确）、商品名称（模糊搜索）、商品分类 ID、上下架状态（1 上架 / 0 下架）、最低价格、最高价格。

**返回数据**：商品概览列表，包含名称、价格、状态等基础信息。

### 2. 商品详情（支持批量）

**用途**：获取商品的完整基础信息。

**返回数据包含**：

- 商品 ID、商品名称
- 分类 ID 和分类名称
- 单位（如"盒""瓶"）
- 基础售价
- 库存数量
- 上下架状态
- 配送方式
- 创建/更新时间和操作人
- 商品图片列表

> **注意**：商品详情是**商品维度**的信息（价格、库存、分类），不包含药品专业信息（说明书、适应症、用法等）。需要药品专业信息时要用药品详情工具。

### 3. 药品详情（支持批量）

**用途**：获取药品的专业说明书信息。

**返回数据包含**：

- 药品通用名、品牌
- 成分、性状
- 功能主治（功效）
- 用法用量
- 不良反应
- 注意事项、禁忌
- 包装规格、有效期、贮藏条件
- 生产单位、批准文号、执行标准
- 产地类型（国产/进口）
- 是否处方药、是否外用药
- 温馨提示
- 药品说明书全文

> **注意**：药品详情是**药品专业维度**的信息，不包含价格、库存、上下架状态等商品经营信息。要同时了解商品经营信息和药品专业信息，需要分别调用商品详情和药品详情。

---

## 商品详情 vs 药品详情的区别

| 信息类型      | 商品详情（get_product_detail） | 药品详情（get_drug_detail） |
|-----------|--------------------------|-----------------------|
| 价格        | ✅ 有                      | ❌ 没有                  |
| 库存        | ✅ 有                      | ❌ 没有                  |
| 分类        | ✅ 有                      | ❌ 没有                  |
| 上下架状态     | ✅ 有                      | ❌ 没有                  |
| 功效/适应症    | ❌ 没有                     | ✅ 有                   |
| 用法用量      | ❌ 没有                     | ✅ 有                   |
| 不良反应/禁忌   | ❌ 没有                     | ✅ 有                   |
| 说明书全文     | ❌ 没有                     | ✅ 有                   |
| 生产单位/批准文号 | ❌ 没有                     | ✅ 有                   |
| 是否处方药     | ❌ 没有                     | ✅ 有                   |

## 典型场景与调用方式

### 场景 1：搜索商品

> 用户问法："帮我搜一下名字带'感冒'的商品"、"有没有维生素相关的药？"

调用：`product_tool_agent("搜索名称含'感冒'的商品列表")`

关注返回中的：匹配的商品名称、价格、状态。

---

### 场景 2：查看价格区间商品

> 用户问法："50 块以下有什么药？"、"价格在 100 到 200 之间的商品"

调用：`product_tool_agent("查询价格在 50 元以下的商品列表")`

关注返回中的：符合价格条件的商品列表。

---

### 场景 3：检查库存情况

> 用户问法："感冒灵颗粒还有多少库存？"、"看看这几个商品的库存够不够"

调用：`product_tool_agent("查询名称含'感冒灵颗粒'的商品详情，关注库存数量")`

关注返回中的：stock（库存数量）字段。

---

### 场景 4：查看上下架状态

> 用户问法："哪些商品是下架状态？"、"现在上架的商品有多少？"

调用：`product_tool_agent("查询下架状态的商品列表")`

关注返回中的：下架商品的名称和数量。

---

### 场景 5：对比多个商品

> 用户问法："帮我对比一下商品 2001 和 2003 的价格和库存"

调用：`product_tool_agent("查询商品 ID 为 2001 和 2003 的详情，对比价格和库存")`

关注返回中的：两个商品各自的价格、库存、分类。

---

### 场景 6：查看药品说明书

> 用户问法："这个药有什么禁忌？"、"帮我查一下这个药的用法用量"、"说明书给我看看"

调用：`product_tool_agent("查询商品 ID 为 2001 的药品详情，包括禁忌和用法用量")`

关注返回中的：功效、用法用量、不良反应、禁忌、注意事项。

---

### 场景 7：了解药品资质信息

> 用户问法："这个药是哪个厂家生产的？"、"批准文号是什么？"、"是处方药吗？"

调用：`product_tool_agent("查询商品 ID 为 2001 的药品详情，关注生产单位和批准文号")`

关注返回中的：生产单位、批准文号、是否处方药、执行标准。

---

### 场景 8：同时了解商品和药品信息

> 用户问法："这个商品的全部信息都给我看看，包括说明书"

需要两步：

1. `product_tool_agent("查询商品 ID 为 2001 的商品详情")` → 获取价格、库存、分类
2. `product_tool_agent("查询商品 ID 为 2001 的药品详情")` → 获取说明书、适应症、用法

最终合并两部分信息给用户。

---

### 场景 9：按分类浏览商品

> 用户问法："保健品分类下有哪些商品？"、"分类 ID 是 3 的商品列表"

调用：`product_tool_agent("查询分类 ID 为 3 的商品列表")`

关注返回中的：该分类下的商品名称和数量。

---

## 跨域使用提示

商品工具通常作为**下游工具**被调用，常见入口：

| 来源                 | 如何获取商品 ID        | 调用方式                                         |
|--------------------|------------------|----------------------------------------------|
| 订单详情中的 productInfo | 返回结果内有 productId | `product_tool_agent("查询商品 ID xxx 的详情")`      |
| 售后详情中的 productInfo | 返回结果内有 productId | `product_tool_agent("查询商品 ID xxx 的药品详情")`    |
| 运营分析 - 热销排行        | 返回结果内有商品 ID      | `product_tool_agent("查询商品 ID xxx 的详情，关注库存")` |
| 运营分析 - 退货率         | 返回结果内有商品 ID      | `product_tool_agent("查询商品 ID xxx 的药品详情")`    |

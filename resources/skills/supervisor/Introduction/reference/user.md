# 用户工具使用指南

## 工具概述

`user_tool_agent` 负责所有用户相关查询，内部包含 5 个工具，接收自然语言任务描述。

---

## 工具能力与返回数据

### 1. 用户列表

**用途**：按条件搜索用户，支持分页浏览。

**可用筛选条件**：用户 ID（精确）、用户名（模糊）、昵称（模糊）、角色、状态（启用/禁用）、创建人。

**返回数据**：用户概览列表，包含 ID、真实姓名、昵称、头像、手机号、邮箱、性别、状态。

### 2. 用户详情

**用途**：获取单个用户的完整资料。

**返回数据包含**：

- **概览**：头像、昵称、钱包余额、总订单数、总消费金额
- **基础信息**：用户 ID、真实姓名、手机号、邮箱、性别、身份证号
- **安全信息**：注册时间、最后登录时间、最后登录 IP、最后登录位置、账号状态

### 3. 用户钱包

**用途**：获取用户的钱包余额和状态。

**返回数据包含**：用户 ID、钱包编号、可用余额、累计入账金额、累计支出金额、币种、钱包状态、冻结原因（如有）、冻结时间（如有）。

### 4. 钱包流水

**用途**：分页查询用户的钱包资金变动记录。

**返回数据**：钱包流水明细列表（收入/支出、金额、时间等），支持分页。

### 5. 消费信息

**用途**：分页查询用户的消费记录。

**返回数据**：消费记录列表（消费金额、时间、关联订单等），支持分页。

## 典型场景与调用方式

### 场景 1：搜索用户

> 用户问法："帮我查一下昵称叫小王的用户"、"有没有叫张三的用户？"

调用：`user_tool_agent("查询昵称含'小王'的用户列表")`

关注返回中的：匹配用户的 ID、姓名、手机号、状态。

---

### 场景 2：查看用户完整资料

> 用户问法："用户 1001 的详细信息"、"这个用户的全部资料"

调用：`user_tool_agent("查询用户 ID 为 1001 的详情")`

关注返回中的：基本信息（姓名/手机/邮箱）、订单统计、消费总额、账号状态。

---

### 场景 3：检查用户钱包状态

> 用户问法："这个用户钱包余额多少？"、"用户的钱包有没有被冻结？"

调用：`user_tool_agent("查询用户 ID 为 1001 的钱包信息")`

关注返回中的：可用余额、钱包状态、如有冻结则关注冻结原因。

---

### 场景 4：排查资金流水

> 用户问法："看看这个用户最近的钱包流水"、"用户 1001 有没有异常的资金变动？"

调用：`user_tool_agent("查询用户 ID 为 1001 的钱包流水，第 1 页")`

关注返回中的：收入/支出记录、金额、变动时间。

---

### 场景 5：查看消费行为

> 用户问法："用户 1001 最近消费了多少？"、"看看他的消费记录"

调用：`user_tool_agent("查询用户 ID 为 1001 的消费信息")`

关注返回中的：消费金额、消费时间、关联订单。

---

### 场景 6：筛选特定角色或状态用户

> 用户问法："管理员角色的用户有哪些？"、"被禁用的用户列表"

调用：`user_tool_agent("查询角色为 admin 的用户列表")` 或 `user_tool_agent("查询状态为禁用的用户列表")`

关注返回中的：符合条件的用户及其状态。

---

### 场景 7：用户全面排查

> 用户问法："全面看看用户 1001 的情况，包括钱包和消费"

需要多步：

1. `user_tool_agent("查询用户 ID 为 1001 的详情")` → 获取基础资料和概览
2. `user_tool_agent("查询用户 ID 为 1001 的钱包信息")` → 获取钱包余额和状态
3. `user_tool_agent("查询用户 ID 为 1001 的消费信息")` → 获取消费记录

最终汇总三部分信息给用户。

---

### 场景 8：核验用户安全信息

> 用户问法："这个用户什么时候注册的？最后一次登录是什么时候？"

调用：`user_tool_agent("查询用户 ID 为 1001 的详情，关注注册时间和最后登录信息")`

关注返回中的：注册时间、最后登录时间/IP/位置。

## 跨域使用提示

用户工具通常作为**下游工具**被调用，也可作为**上游工具**触发后续查询：

### 作为下游工具（从其他工具拿到 userId 后调用）

| 来源             | 如何获取用户 ID     | 调用方式                                      |
|----------------|---------------|-------------------------------------------|
| 订单详情的 userInfo | 返回结果内有 userId | `user_tool_agent("查询用户 ID xxx 的详情")`      |
| 售后详情的 userId   | 返回结果内有 userId | `user_tool_agent("查询用户 ID xxx 的钱包和消费信息")` |

### 作为上游工具（查到用户后需要更多关联数据）

| 用户后续问法      | 下一步调用                              |
|-------------|------------------------------------|
| "这个用户买过什么？" | order_tool_agent 用用户信息查订单          |
| "他有没有提过售后？" | after_sale_tool_agent 按用户 ID 查售后列表 |

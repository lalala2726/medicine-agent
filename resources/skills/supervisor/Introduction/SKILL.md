# Supervisor 子工具调用指南

## 技能目标

帮助 supervisor_agent 正确使用 5 个子代理工具，重点解决两个问题：

1. **单工具使用**：每个子代理能做什么、参数怎么传、返回哪些数据、**不返回**哪些数据
2. **跨域协作**：一个工具的返回结果包含其他域的 ID 时，如何接力调用下一个工具获取完整信息

## 子代理工具能力总览

| 子代理                | 内部工具               | 核心能力                          |
|---------------------|--------------------|---------------------------------|
| order_tool_agent    | 订单列表 / 订单详情 / 流程时间线 / 发货记录 | 订单筛选浏览、订单完整信息查询、状态追踪、物流履约           |
| product_tool_agent  | 商品列表 / 商品详情 / 药品详情          | 商品搜索筛选、商品基础信息、药品说明书与专业信息         |
| user_tool_agent     | 用户列表 / 用户详情 / 钱包 / 钱包流水 / 消费信息 | 用户搜索筛选、用户资料核验、资金状态与消费行为排查       |
| after_sale_tool_agent | 售后列表 / 售后详情                  | 售后申请筛选、处理进度追踪、售后全链路信息             |
| analytics_tool_agent | 运营总览 / 订单趋势 / 状态分布 / 支付分布 / 热销排行 / 退货率 | 经营数据总览、趋势分析、排行与分布统计 |

## 跨域协作指南（核心）

当用户的需求**跨越多个业务域**时（如"查订单里的商品详细信息"），需要多工具接力调用：

- [跨域协作操作手册](reference/cross_agent.md) — 12 种常见跨域场景的完整分步操作指引

### 核心原则速记

1. **ID 是跨域的钥匙**：拿到 productId → 调商品工具；拿到 userId → 调用户工具；拿到 orderId → 调订单工具
2. **摘要 ≠ 详情**：订单/售后中的商品信息只有名称、数量、单价，不含库存/分类/说明书
3. **用户要"详细"就要补查**：听到"详细信息""完整信息""更多信息"等关键词，必须跨域获取
4. **传 ID 要明确**：调下游工具时，在任务描述中写清具体的 ID 值

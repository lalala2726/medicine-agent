# 自适应跨域联动技能（Adaptive Agent）

## 技能目标
指导你在面对涉及多个业务域（如订单+用户、商品+运营等）的复杂请求时，如何正确地进行“意图拆解”、“跨域 ID 接力”以及最终的“数据聚合”。

## 跨域接力核心原则
在所有涉及跨域关联查询的场景下，**ID是唯一的信任凭证**。
- 【商品域】核心凭证：`productId`、`categoryId`
- 【订单域】核心凭证：`orderId`、`userId`、`productId`
- 【用户域】核心凭证：`userId`
- 【售后域】核心凭证：`afterSaleId`、`orderId`

**绝对禁止**：在未获取有效 ID 的情况下，直接拿用户姓名、商品名称等文本字段去跨域系统硬猜或拼接。必须先调用A域工具拿到准确ID，再用该ID去调用B域工具。

## 典型操作示例与示例数据

### 场景一：订单到用户的跨域查询
**用户问题**：“查一下最新的一个订单，并告诉我那个买家的钱包余额和消费偏好。”

**错误做法**：直接调用用户画像工具，不知道查谁。
**正确执行步骤**：
1. **获取订单（获取凭证）**：
   - 调用工具：`get_order_list(limit=1, sort="desc")`
   - 示例返回数据：`[{"orderId": "ORD88992", "userId": "U10045", "status": "shipped"}]`
   - **关键动作**：提取出 `userId` 为 `U10045`。
2. **获取用户（消费凭证）**：
   - 调用工具：`get_admin_user_wallet(userId="U10045")` 和 `get_admin_user_detail(userId="U10045")`
   - 示例返回数据：`{"balance": 150.50, "status": "active"}`，`{"tags": ["高净值用户", "复购率高"]}`
3. **数据聚合输出**：
   - “最新的订单（单号 ORD88992）目前已发货。该订单的买家钱包余额为 150.50 元，系统画像显示其为高净值用户且复购率较高。”

### 场景二：售后到订单，再到商品的跨域关联
**用户问题**：“昨天被拒绝退款的那个售后单，对应的是什么商品？这个商品现在的库存还有多少？”

**正确执行步骤**：
1. **锁定售后单（获取第一层凭证）**：
   - 调用工具：`get_admin_after_sale_list(status="rejected", date="yesterday")`
   - 示例返回数据：`[{"afterSaleId": "AS9901", "orderId": "ORD5566", "reason": "包装已拆"}]`
   - **关键动作**：提取出 `orderId` 为 `ORD5566`。
2. **锁定订单（获取第二层凭证）**：
   - 调用工具：`get_orders_detail(orderId="ORD5566")`
   - 示例返回数据：`{"orderId": "ORD5566", "productId": "P88123", "quantity": 2}`
   - **关键动作**：提取出 `productId` 为 `P88123`。
3. **查询商品（消费最终凭证）**：
   - 调用工具：`get_product_detail(productId="P88123")`
   - 示例返回数据：`{"productName": "感冒灵胶囊", "stock": 500, "price": 28.0}`
4. **数据聚合输出**：
   - “昨天被拒的售后单（AS9901，拒绝原因：包装已拆）关联的商品是‘感冒灵胶囊’。目前该商品库存充足，还有 500 件。”

### 场景三：无果时的熔断处理
当某一环节的查询未返回预期 ID 时，应立即停止后续无关工具的调用，向用户诚实报告中断节点。
- 例如：售后列表为空，没有拿到 `orderId`。
- **输出示例**：“未查到昨天被拒绝的售后记录，因此无法为您继续查询关联的商品信息。”

# 售后管理能力

## 能力概述

系统支持管理端售后查询能力，覆盖售后列表与售后详情，帮助运营和客服快速定位异常订单、跟踪售后处理进度。

---

## 售后列表查询

按多种条件分页查询售后申请列表。

### 支持的筛选维度

| 筛选维度  | 说明                                                                 |
|-------|--------------------------------------------------------------------|
| 售后类型  | REFUND_ONLY / RETURN_REFUND / EXCHANGE                             |
| 售后状态  | PENDING / APPROVED / REJECTED / PROCESSING / COMPLETED / CANCELLED |
| 订单编号  | 精确匹配                                                               |
| 用户 ID | 精确匹配                                                               |
| 申请原因  | 例如 DAMAGED                                                         |
| 分页参数  | page_num / page_size 分页查询                                          |

### 使用示例

> "查一下待处理的售后申请"
> "按订单号 O20251108123456789012 查售后记录"
> "查用户 1001 的售后申请列表"

---

## 售后详情查询

根据 `after_sale_id` 查询单笔售后详细信息（如处理状态、申请原因、处理进度等）。

### 使用示例

> "查看售后申请 30001 的详情"
> "帮我看一下这笔售后当前处理到哪一步了"

---

## 常见业务场景

| 场景       | 推荐问法                                |
|----------|-------------------------------------|
